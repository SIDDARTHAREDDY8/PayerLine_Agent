PayerLine calls the insurer
and catches its mistakes.

A miniature of VoiceAdmin's core loop: an agent phones a payer, verifies eligibility, and returns clean, EHR-ready data — with a reliability layer that flags a rep's bad numbers before they reach billing, then routes only the risky calls to a human. Every call is generated live and scored against a known ground truth.

Which calls need a human?

python review_queue.py

You can't review millions of calls a year — and you can't blindly auto-post data that might be wrong. So every finished call is scored per-field and routed: auto-post when it's confident and consistent, human review when it isn't. Humans only touch the fraction the system is unsure about.

Stress-tested through a phone line

python stress_eval.py

The transcript above is clean text — so I measured what happens when it isn't. Every rep answer is spoken, degraded to a narrowband, noisy telephone channel (300–3400 Hz, ~12 dB SNR), and transcribed back with a local speech-to-text model — the exact path a production agent hears — then re-extracted. Free and offline: say + faster-whisper, no API for the audio path.

100→93%field accuracy: clean text → through phone + STT

What voice got wrong — and what happened to it

plan type · PPO heard as PTO
rode in a call already flagged for an impossible deductible — held back, never posted
↻ held
specialist copay · $40 heard as $1.40
same call, same hold — re-verify on a fresh line before anything reaches billing
↻ held

Honestly: the verifier catches structural errors with certainty — deductible-met > deductible, inactive-with-copays, missing fields. A plausible-but-wrong value that breaks no rule is only stopped when it rides in an already-flagged call or drops the field's confidence below the auto-post bar. Closing that gap — re-confirming low-confidence numbers on the call — is the roadmap. The point stands: voice introduces errors, and the deterministic layer keeps the structural ones out of the EHR, measured end to end.

Honestly scoped: the agent speaks in real voices in the live demo; what's still stubbed is telephony — the same loop drops onto Twilio / LiveKit with STT, and call.py is the seam. The payer is simulated so accuracy is measurable, and the verification rules are hand-written today — the next step is learning them from labeled call outcomes. I scoped this to the reasoning + reliability layer on purpose: that's the hard part.
